SecuritySécurité

Every school, completely isolatedChaque école, totalement isolée

Multi-tenant by design — your data never mixes with another school's, ever.Multi-tenant par conception — vos données ne se mélangent jamais avec celles d'une autre école.

01

Tenant isolationIsolation par tenant

Every query is filtered by school_id — no cross-school data access is possible.Chaque requête est filtrée par school_id — aucun accès aux données d'une autre école.

02

JWT authenticationAuthentification JWT

Signed, stateless tokens carry role and tenant context on every request.Des jetons signés et sans état transportent le rôle et le contexte tenant.

03

Rate limitingLimitation de débit

Requests are throttled per IP and endpoint to prevent abuse.Les requêtes sont limitées par IP et par endpoint pour prévenir les abus.

04

Hardened headersEn-têtes durcis

Helmet.js applies secure HTTP headers against XSS and clickjacking.Helmet.js applique des en-têtes HTTP sécurisés contre le XSS et le clickjacking.

05

Subscription checksVérification d'abonnement

Access is blocked instantly if a school's subscription is suspended.L'accès est bloqué instantanément si l'abonnement est suspendu.

06

Encrypted in transitChiffrement en transit

All traffic is encrypted with TLS — no sensitive data ever travels in plain text.Tout le trafic est chiffré en TLS — aucune donnée sensible en clair.
